Задачи:

- добавить full-screen canvas;
- подключить Three.js renderer, scene, camera, lights;
- подключить OrbitControls;

Критерий готовности:

- камера управляется мышью;
- production build собирается без ошибок.

Примечание: production build на этапе 1 предупреждает о JS chunk чуть больше 500 KB из-за Three.js. Это не блокирует MVP; оптимизировать code splitting позже, когда появятся реальные модули физики/UI.

## Этап 2. Модель данных и базовая CPU-физика

Статус: выполнен.

Цель: симуляция нескольких тел на CPU в браузере.

Задачи:

- описать типы тел: id, name, mass, radius, position, velocity, color;
- добавить поле `pinned/static` для тел;
- хранить initial state отдельно от runtime state;
- реализовать ньютоновскую гравитацию;
- добавить softening factor;
- реализовать velocity Verlet или leapfrog;
- добавить fixed timestep;
- добавить старт, паузу, продолжение и сброс;
- ввести интерфейс `PhysicsBackend`.

Критерий готовности:

- два и более тела притягиваются друг к другу;
- сброс возвращает сцену к исходному состоянию;
- изменение `G` влияет на движение;
- CPU backend используется как эталон корректности.

Примечание: реализован `PhysicsBackend`, `CpuPhysicsBackend`, velocity Verlet, fixed timestep, softening factor, pinned/static тела и тесты физики.

Follow-up: стартовая сцена заменена на Солнечную систему по NASA/JPL Horizons `DE441` на `2026-06-16 00:00:00 TDB`. Для устойчивости WebGPU/CPU расчетов используются единицы: расстояние в млн км, время в днях, масса в массах Солнца.

## Этап 3. WebGPU compute backend

Статус: реализован, требуется браузерная runtime-проверка на WebGPU-устройстве.

Цель: перенести расчет гравитации на GPU пользователя там, где WebGPU доступен.

Задачи:

- реализовать инициализацию WebGPU: adapter, device, limits, error handling;
- описать layout GPU buffers: position, velocity, mass, acceleration;
- написать WGSL compute shader для попарного расчета ускорений;
- реализовать fixed timestep на WebGPU backend;
- синхронизировать результат с рендером без лишнего readback на CPU;
- добавить переключатель backend: Auto, CPU, WebGPU;
- показывать активный backend в UI;
- добавить fallback на CPU/Worker при ошибке WebGPU.

Критерий готовности:

- на браузере с WebGPU симуляция работает через GPU compute;
- на браузере без WebGPU приложение продолжает работать через fallback;
- результаты CPU и WebGPU совпадают в допустимой численной погрешности на малой сцене.

Примечание: реализован `WebGpuPhysicsBackend` с WGSL compute shader, `vec4<f32>` buffer layout, 4-pass velocity Verlet, readback для Three.js/WebGL, выбором backend `Auto/CPU/WebGPU`, fallback на CPU и обработкой `device.lost`. Автоматические Node-тесты покрывают CPU-эталон; WebGPU runtime нужно проверять в браузере.

## Этап 4. UI управления сценой

Статус: частично реализован.

Цель: пользователь может создавать и редактировать систему.

Задачи:

- панель управления симуляцией;
- slider/input для `G`;
- slider/input для скорости времени;
- controls для ускорения/замедления симуляции;
- input/select для fixed timestep с безопасными пределами;
- отображение текущего timestep;
- отображение масштаба сцены;
- кнопки start/pause/resume/reset/clear;
- список тел;
- выбор тела кликом;
- инспектор выбранного тела;
- автогенерация названий тел по словарю;
- ручное редактирование названия тела;
- редактирование цвета через RGB/hex и палитру;
- редактирование массы в экспоненциальной записи;
- редактирование радиуса или визуального размера;
- редактирование координат `x`, `y`, `z`;
- редактирование скорости `vx`, `vy`, `vz`;
- checkbox `Pinned` / `Static body`;
- readonly-отображение ускорения `ax`, `ay`, `az`;
- добавление тела из пресетов.

Критерий готовности:

- пользователь может собрать простую систему без изменения кода.
- выбранное тело можно отредактировать через инспектор.
- reset восстанавливает параметры симуляции и тела к начальному состоянию.
- pinned-тело можно включить/выключить через инспектор.

Примечание: реализованы выбор тела кликом, габаритный corner-box marker, инспектор имени/цвета/массы/radius/координат/скоростей/ускорений/pinned, добавление, удаление, `Clear`, список тел снизу слева и фокус камеры.

Follow-up: сцена переведена на привычную плоскость XOY с осью Z вверх; `Focus` работает как toggle-follow выбранного тела; редактирование параметров доступно только когда симуляция не running и больше не сбрасывает elapsed time.

Осталось по этапу:

- добавить явный input/select для fixed timestep;
- добавить отображение масштаба сцены;
- добавить цветовую палитру быстрых цветов кроме native color picker;
- добавить пресеты добавления тел.

## Этап 5. 3D-взаимодействие

Статус: частично реализован.

Цель: удобная работа с объектами в сцене.

Задачи:

- raycasting для выбора тел мышью;
- подсветка выбранного тела;
- TransformControls или собственные gizmo для перемещения;
- фокус камеры на выбранном теле;
- отображение векторов скорости/ускорения;
- шлейфы/траектории тел цветом соответствующего тела;
- настройка длины и видимости шлейфов;
- очистка шлейфов при reset/clear;
- защита от чрезмерного расхода памяти на шлейфы.

Критерий готовности:

- объект можно выбрать и выставить в сцене мышью.
- пользователь видит историю движения тел в виде шлейфов.

Примечание: реализованы raycasting-выбор, подсветка выбранного тела через corner-box marker, фокус/follow выбранного тела, шлейфы тел цветом соответствующего тела, настройка видимости и длины шлейфов, infinite trail mode без обрезки истории, очистка при reset/clear и ограничение памяти через bounded history.
